### **미션 구조 (개조식 정리)**



목표: AI 비서 패키지 개발 — 특정 업무(예: 회의록 요약, 이메일 작성) 수행 AI 설계·검증

산출물: 3종 문서 (모델 비교 보고서 / 시스템 설계서 / 실행 로그)



**1. 모델 비교 보고서**

동일 업무를 3개 이상 AI에 테스트

평가 기준: 한국어 자연도, 형식 준수, 정책/안전성, 맥락 유지, 톤, 속도·안정성

결론: 업무 적합 최종 모델 선정 및 근거 명시



**2. 시스템 설계서**

페르소나 정의: 역할·경력·전문성 부여 (예: "10년 차 PM")

시스템 프롬프트: 말투, 제약사항, 모르면 되묻기 등 규칙 명시

Few-shot 예시: 질문-답변 세트 3개 내외 사전 제시

버전 개선: v1 → v2 프롬프트 고도화 (비교 근거 포함)



**3. 실행 로그 및 검증**

10턴 이상 대화: 실제 업무 수행 시나리오

조건 변경 테스트: 대화 중 조건 변경 시 대응력 확인

환각 검증: 난이도 높은 질문 5개로 사실관계 오류 점검



### **모델 비교 평가 보고서 (요약)**



**1. 평가 결과표 (5점 만점)**

모델정확성공감도가독성대안제시평균A. Claude2.01.04.01.52.1B. Gemini4.35.03.74.74.4C. GPT4.03.74.35.04.3



**2. 모델별 핵심 평가**

A. Claude — 치명적 결함 발견



역할 미이행: 시스템 프롬프트("구청 민원실 AI 비서")를 무시하고 3개 케이스 전부 "학습 도우미 AI 튜터"로 자기소개 → 페르소나 완전 실패

공감 표현 없이 즉시 외부 연결(표 형태)로 회피 → 민원인 감정 케어 전무

가독성(표, 이모지)은 우수하나, 업무 자체를 수행하지 않음

결론: 이번 테스트 조건에서는 시스템 프롬프트 미적용 오류로 순위 산정 자체가 무의미. 재테스트 필요



B. Gemini — 공감·실행력 최상위



case1: 소음 상황 즉각 공감 → 확인질문(위치, 접수번호) → 부서 연계 조치까지 자연스럽게 전개

case3: 법적 권한 한계를 솔직히 인정하면서도 실질적 대안(이웃대화, 관리사무소, 동물보호센터) 제시

다만 case1에서 "직접 연락하겠다"는 표현은 AI가 실행 불가능한 액션을 암시 → 환각성 과장 주의



C. GPT — 구조화·행동지침 최상위



모든 케이스에서 단계별(1\~5단계) 구조 + 확인질문 + 예시 문구(민원 작성용 멘트) 제공

사실관계(구청 권한 범위, 법적 한계) 가장 신중하고 정확하게 서술

다만 답변 길이가 과도하게 김 → 실전 민원 챗봇으로는 정보과잉, 가독 피로도 우려



**3. 종합 순위 (Claude 제외)**

1위 GPT > 2위 Gemini (근소한 차이, 용도별 상이)



정확성·신중함 중시 → GPT

공감·간결한 실행력 중시 → Gemini

**## 2. Case별 평가 요약**



\### Case 1 — 도로공사 소음 (감정형 불만)

\- \*\*C > B > A\*\*

\- ChatGPT: 야간/주간 구분, 재민원 경로, 증거 확보 안내까지 가장 구체적

\- Gemini: 공감 최고점이나 "직접 부서에 연락하겠다"는 표현이 AI 권한 초과로 감점

\- Claude: 역할 거부로 정확성·공감 모두 최하점



\### Case 2 — 지원금 서류 문의 (정보 부족형)

\- \*\*C ≳ B > A\*\* (C·B 근소한 차이)

\- 둘 다 "확인 질문 먼저" 규칙 준수 + 일반 서류 목록 병행 제시

\- Claude: 확인 질문 규칙 미준수, 링크 나열에 그침



\### Case 3 — 옆집 강아지 소음 (불가능한 요구형)

\- \*\*C > B > A\*\*

\- Gemini: 동물보호법 근거 제시 등 정확성·공감 모두 우수

\- ChatGPT: 5단계 구조 + 상황별 분기(학대·방임 시 대응) 가장 체계적

\- Claude: 동일 패턴(역할 미수행) 반복



\---



**## 3. 핵심 결함 — Claude (A) 공통 지적사항**

\- \*\*시스템 프롬프트("구청 민원 AI 비서") 완전 무시\*\* → 전 케이스에서 "학습 도우미"로 자기소개

\- 정중한 어투·가독성(표 형식)은 우수하나 \*\*실제 민원 해결 기능 미수행\*\*

\- 3개 평가(GPT/Gemini/Claude 자체 평가) 모두 동일하게 "평가 자체가 무의미"라고 지적 → \*\*재현성 있는 결함\*\*



**## 4. 최종 제언**

\- \*\*실제 도입 시 조합 전략\*\*: ChatGPT(정보 구체성) + Gemini(공감력)의 장점 결합 프롬프트 설계 권장

\- \*\*Claude 재검증 필수\*\*: 시스템 프롬프트 강제 적용(persona lock) 여부를 최상단에 명시 후 재테스트

\- 조아패션 프로젝트와 마찬가지로, \*\*모델의 역할 고정력(persona adherence)\*\* 자체가 별도 평가축으로 추가될 필요 있음



### **# 3단계: 시스템 설계 및 고도화 (요약)**



**## 1. 개요**

\- \*\*목표\*\*: 시스템 프롬프트 v2 + Few-shot 3개 → 최종 운영 프롬프트 완성

\- \*\*배경\*\*: 2단계 평가에서 Claude(v1 프롬프트)가 페르소나 미이행 → 강제력 있는 v2 설계 필요



\---



**## 2. 시스템 프롬프트 v1 → v2 개선 내역**



| 구분 | v1 | v2 |

|---|---|---|

| 정체성 | "구청 민원실 AI 상담 비서" (단순 역할명) | 이름("구청이") + 직책 + 경력(5년) 부여 → 역할 고정력 강화 |

| 답변 구조 | 명시 없음 | 4단계 강제(공감→상황파악→해결방법→다음단계) |

| 권한 범위 | 원칙만 명시("권한 밖은 솔직히") | ✅할 수 있는 것 / ❌할 수 없는 것 \*\*구체적 항목 나열\*\* |

| 금지사항 | 없음 | 6개 항목 명시(법적조언, 확답 약속, 개인정보 공개 등) |

| 분량 제한 | 없음 | 100\~600자 구간별 기준 + 최대치 설정 |

| 정보부족 대응 | "확인 질문 먼저" 원칙만 | 필수/선택 정보 구분 + 질문 예시 문구 제공 |



\*\*개선 핵심\*\*: 원칙 나열(v1) → 실행 가능한 세부 규칙(v2)으로 전환, 특히 역할 이탈 방지를 위한 정체성 강화가 두드러짐

**## 3. Few-shot 예시 구성 (3종)**



| 케이스 | 유형 | 평가 포인트 |

|---|---|---|

| 예시1 | 감정형 불만 (도로공사 소음, 3개월 지속) | 공감 표현, 확인질문 2개, 3단계 안내, 보상 불확실성 솔직 고지 |

| 예시2 | 모호한 정보 (지원금 종류 미상) | 필수정보 3개+선택정보 2개 구조화 질문, 환각 방지(확정 정보 미제시) |

| 예시3 | 불가능한 요구 (이웃 소음, 경찰신고 무효) | 권한범위 3단계 구분(구청/경찰/법원), 대안 3가지 + 추천 순서 |



\*\*공통 설계 원칙\*\*: 모든 예시가 "공감 → 권한 설명 → 구체적 대안 → 연락처" 흐름을 일관되게 따름
